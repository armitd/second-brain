#### **Part 1: Essential Skills Most Users Never Install (1–10)**

**01. PDF Processing Skill** 

Read PDFs, extract tables, fill forms, merge documents, and split files. This is the single highest-utility Skill for anyone who works with documents. Instead of using three different online tools to handle PDFs, Claude does it all natively. 

Most users upload a PDF and ask Claude to "read it." The PDF Skill does full extraction with structured output, table parsing, and batch processing. 

**02. DOCX Skill** 

Create and edit Word documents with proper formatting, tracked changes, comments, and professional layouts. Not the generic text that Claude outputs in chat - actual .docx files with headers, page numbers, and real formatting. 

Most users copy Claude's text output and paste it into Word manually. The DOCX Skill creates the finished document directly. 

**03. PPTX Skill** 

Generate presentation slide decks from natural language descriptions. Layouts, charts, speaker notes, and consistent design - all produced automatically. Say "create a 12-slide pitch deck for my startup" and get an actual PowerPoint file. 

Most users build slides manually and paste Claude's text in one slide at a time. This is dramatically faster. 

**04. XLSX Skill** 

Create spreadsheets with formulas, analysis, formatting, and charts using plain English instructions. Ask Claude to "build a financial model with revenue projections for 12 months" and get a working spreadsheet with real formulas. 

Most users ask Claude for the formula and then type it into Excel themselves. The XLSX Skill eliminates the middleman. 

**05. Frontend Design Skill** 

Escape generic "AI slop" aesthetics. This Skill teaches Claude real design systems - bold typography, distinctive color palettes, professional layouts. Over 277,000 installs for a reason. If you build anything with a user interface, this Skill is non-negotiable. 

**06. Canvas Design Skill** 

Social media graphics, posters, covers, and visual content generated from text descriptions. Output as PNG or PDF. If you create any kind of visual content, this eliminates the need for a separate design tool for most tasks. 

**07. SEO Skill** 

Full website audits, schema validation, keyword analysis, and content optimization. Twelve sub-skills covering every part of SEO. Most users ask Claude for generic SEO advice. The SEO Skill runs a systematic audit with specific, actionable findings. 

**08. Brand Guidelines Skill** 

Encode your brand - colors, fonts, tone of voice, messaging rules - into a Skill that automatically applies everywhere. Every document, every design, every piece of content follows your brand standards without you reminding Claude every time. 

**09. Marketing Skills Bundle** 

Twenty-plus Skills in one package covering CRO, copywriting, SEO, email sequences, and growth strategies. If you work in marketing, this single installation gives Claude specialized knowledge across your entire discipline. 

**10. Obsidian Skills** 

Built by Obsidian's CEO. Automatic tagging, linking, and organization of your Obsidian vault. If you use Obsidian for notes and research, this Skill turns Claude into a vault manager that keeps everything organized and interconnected. 

#### **Part 2: Workflows That Save Hours Every Week (11–20)**

Skills teach Claude how. Workflows teach Claude when and in what order. 

**11. The Morning Briefing Workflow** 

Set this up once and run it every morning: 

"Check my email for anything urgent. Summarize overnight Slack messages. Pull my calendar for today. Identify the three most important things I need to focus on. Save everything to a daily briefing file." 

Run this as a scheduled task at 7am. You wake up to a complete situation report every single day. 

**12. The Content Production Pipeline** 

A four-step workflow that takes any topic from idea to published article: 

Step 1: Research the topic and create a brief with key findings Step 2: Create a detailed outline with headline, sections, and key points Step 3: Write the full draft following the outline Step 4: Edit and polish to publication quality 

Run all four steps in one Cowork session. A complete article in under 30 minutes. 

**13. The Weekly Report Compiler** 

Every Friday at 5pm: 

"Pull all my completed tasks from this week. Summarize key metrics from [source]. Compile wins, blockers, and next week's priorities into a formatted report. Save to /Reports." 

Your weekly report writes itself while you are wrapping up for the day. 

**14. The Email Response Workflow** 

For every email that needs a reply: 

"Read this email. Identify the sender's request. Draft a response that is professional, under five sentences, and includes a clear next step. Match the formality level of the sender." 

Process twenty emails in the time it normally takes to write three. 

**15. The Competitive Intelligence Workflow** 

Monthly recurring: 

"Research [competitor names]. Check for new product launches, pricing changes, feature updates, and public statements. Compare to our current position. Save a competitive intelligence brief to /Intelligence." 

Stay informed about your competition without spending hours on manual research. 

**16. The File Organization Workflow** 

Weekly: 

"Scan my Downloads folder. Sort files by type - documents to /Docs, images to /Images, spreadsheets to /Data. Rename files with a consistent naming convention. Delete anything older than 60 days that is not marked important." 

Your digital workspace stays clean automatically. 

**17. The Client Onboarding Workflow** 

For every new client: 

"Create a new folder called [Client Name]. Generate a welcome document from my template. Create a project timeline based on [project scope]. Draft an onboarding email introducing the process and next steps. Save everything to the client folder." 

New clients get a professional onboarding experience in minutes instead of hours. 

**18. The Learning Summary Workflow** 

After consuming any educational content: 

"Read my notes on [topic]. Extract the key concepts. Identify how each concept applies to my current work. Create a one-page summary with action items. Save to /Learning." 

Every book, course, and article gets distilled into actionable knowledge. 

**19. The Social Media Scheduling Workflow** 

Weekly: 

"Based on my content calendar, draft posts for each platform for the coming week. Format each post for its platform - Twitter under 280 characters, LinkedIn under 200 words, Instagram with hashtags. Save all drafts to /Social organized by day." 

A full week of social media content prepared in one session. 

**20. The Invoice and Receipt Workflow** 

Monthly: 

"Process all receipt images in /Receipts. Extract date, vendor, amount, and category. Create a spreadsheet with all transactions sorted by category. Calculate totals. Flag anything unusual. Save to /Finance." 

Monthly expense tracking that would take an hour is done in minutes. 

#### **Part 3: Power User Configurations (21–30)**

These are the setups that separate beginners from power users. 

**21. The Master Context File** 

Create a file called [context.md](http://context.md/) that contains everything Claude needs to know about you: 

* Who you are and what you do
* Your audience and what they care about
* Your tone of voice and communication style
* Your current projects and priorities
* Topics you cover and topics you avoid
* Examples of your best work

Add "Read [context.md](http://context.md/) fully before starting any task" to every Skill and workflow. This single file eliminates the need to re-explain yourself in every conversation. 

**22. The [CLAUDE.md](http://claude.md/) File for Code Projects** 

If you use Claude Code, create a [CLAUDE.md](http://claude.md/) file in every project root. Include your tech stack, coding conventions, folder structure, and preferences. Claude reads this before every session and follows it automatically. 

Every time you correct Claude twice on the same thing, add it here. The file gets smarter over time. 

**23. The Prompt Library** 

Create a folder called /Prompts and save your best-performing prompts as individual files. [Content-research-prompt.md](http://content-research-prompt.md/). [Code-review-prompt.md](http://code-review-prompt.md/). [Email-draft-prompt.md](http://email-draft-prompt.md/). Instead of writing prompts from scratch every time, reference your saved library. 

**24. The Template System** 

Build templates for every repeating output: 

* Report template with your standard sections and formatting
* Email templates for common scenarios (outreach, follow-up, decline)
* Document templates with your brand formatting
* Slide deck templates with your standard structure

Upload these to a Claude Project and reference them when needed. "Use my report template and fill it with this week's data." 

**25. The Evaluation Checklist** 

Create a quality checklist that Claude runs against every output before delivering it to you: 

Before delivering any output, verify:

- Does it follow the instructions completely?

- Is it the correct format and length?

- Are all claims specific (numbers, names, dates) rather than vague?

- Does it match the required tone?

- Would I be comfortable sharing this with a client without edits?

If any answer is no, fix it before presenting. 

Add this to the end of every Skill. Quality goes up immediately. 

**26. The Error Recovery Protocol** 

When Claude gets something wrong, most users just correct it in the moment and move on. Power users update their Skills: 

"Add a rule: never use passive voice in marketing copy." "Add a rule: always include the source URL when citing statistics." "Add a rule: check the current date before referencing any time-sensitive information." 

Every mistake becomes a permanent fix. 

**27. The Multi-Project Workspace** 

Set up separate Claude Projects for each major area of your work. Each project has its own system prompt, knowledge files, and conversation history. Switch between projects when you switch between work modes. 

This keeps context clean and outputs relevant. 

**28. The Connector Stack** 

Connect every app you use daily. Gmail for emails. Google Calendar for scheduling. Slack for communications. Google Drive for files. Notion for knowledge management. 

The more Claude can access directly, the less time you spend being the middleman between your tools and your AI. 

**29. The Feedback Loop System** 

Every Friday, spend fifteen minutes reviewing Claude's outputs from the week: 

* What output did not meet my standard?
* What instruction change would have fixed it?
* What new task could I add to my workflow next week?

Update your Skills and workflows based on the answers. This is the compounding effect that most users miss entirely. 

**30. The Skill Builder Workflow** 

Use the Skill Creator meta-skill to build new Skills on demand. Describe a workflow you do manually and let Claude convert it into a permanent Skill: 

"I currently do X manually every week. Turn this into a Skill that Claude can execute automatically with consistent quality." 

Every repetitive task becomes an automated Skill. Your system grows more powerful every single week.
