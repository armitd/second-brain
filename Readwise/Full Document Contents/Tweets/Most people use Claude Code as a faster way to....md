## Full Document
Most people use Claude Code as a faster way to type. One prompt, one output, one shot at the answer. That's why most of what comes out is AI slop.

Gabor's approach is different. He set up 21 specialized agents inside Claude Code, structured exactly like a real software company.

The roles he runs:

* System analyst (writes specs, the most important one)
* Product spec architect (audits whether specs are clear)
* CTO agent (strategic technical decisions)
* UX flow architect (designs clickable prototypes)
* Multiple designer agents (visuals and brand)
* Brand agent (consistency across surfaces)
* Implementation agents (the actual coders)
* Code maintainability agent (prevents spaghetti)
* Performance agent (optimization)
* Test architect (quality gates)
* Product council (data security and privacy)

That's not a prompt collection. That's an org chart.

The reframe most PMs are missing: Claude Code becomes something completely different the moment you treat it like an org you're staffing. Single-prompt thinking caps your output at whatever one model can do in one shot. Multi-agent thinking simulates the entire production pipeline of a software team.

Watch the workflow difference.

Single agent: "Build me a feature." Output is generic, untested, ignores your architecture. Fragile.

21-agent team: System analyst writes spec. Spec architect reviews clarity. UX flow architect designs prototype. CTO agent vets approach. Designers handle visuals. Implementers code. Test architect validates. Performance agent optimizes. Product council audits data handling.

Same final deliverable. Completely different reliability.

Gabor used this setup to ship a hockey rules AI app to TestFlight. Not a prototype, a live app, solo. Claude Code orchestrating Confluence, Figma, Jira, and the iOS Simulator across the entire workflow.

The hidden insight from this episode...

The system analyst is the bottleneck agent. Before any code gets written, it decides whether your spec is even buildable. Gabor spends real time defining what "good" looks like for that one agent, because every downstream agent inherits its quality. Garbage spec in, 21 agents producing garbage out.

That's the unlock.

You don't need 21 agents on day one. You need to stop thinking about Claude Code as a chat box and start thinking about it as an org you're staffing. Which roles do you need? What does "good" look like for each? How do they hand off work?

That mental shift is what separates PMs shipping real apps from PMs producing prompts that go nowhere.
