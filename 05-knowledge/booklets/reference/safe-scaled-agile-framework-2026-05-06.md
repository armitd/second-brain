# SAFe 6.0 — Scaled Agile Framework

## Quick Summary

SAFe (Scaled Agile Framework) is the most widely adopted enterprise agile scaling framework. Unlike DA (Disciplined Agile), which is a toolkit for choosing your way of working, SAFe is a prescriptive, structured framework that defines roles, ceremonies, artefacts, and governance for scaling agile across multiple teams, programmes, and portfolios. Version 6.0 added an "AI-Empowered" focus, updated flow principles, and strengthened the built-in quality practices.

## Key Insights

- **Four configurations — start with Essential, grow to Full:** Essential SAFe is the entry point (Agile Release Trains + Team and Technical Agility + Agile Product Delivery). Large Solution adds multi-team coordination. Portfolio SAFe adds investment funding and lean governance. Full SAFe includes all seven core competencies of the Lean Enterprise. Most organisations start Essential and expand only as needed.

- **The Agile Release Train (ART) is the central unit:** An ART is a long-lived team-of-teams (50–125 people) that plans, commits, and delivers together on a fixed cadence (Program Increment = 8–12 weeks). PI Planning is the flagship ceremony — all ART members align face-to-face (or virtually) at the start of each PI to build a shared plan. This is SAFe's most distinctive and most contested feature.

- **SAFe 6.0 core values:** Alignment, Transparency, Respect for People, Relentless Improvement. Eight SAFe principles underpin the framework (economic view, systems thinking, assume variability, build incrementally, base milestones on objective evaluation, visualise and limit WIP, apply cadence, unlock intrinsic motivation). Principle #6 updated in 6.0 to include eight "flow accelerators."

- **SAFe vs DA contrast:** SAFe is top-down and prescriptive (tells you *what* to do); DA is bottom-up and contextual (helps you *choose* what to do). SAFe suits organisations that need structure imposed at scale from the outset. DA suits organisations with existing agile experience that want to contextualise rather than standardise.

## The Big Picture — Four Levels

| Level | Focus | Key Artefact |
|---|---|---|
| **Team** | Sprint-level delivery, technical practices | Team backlog, definition of done |
| **Programme (ART)** | PI Planning, programme execution, system demo | Program Increment (PI) plan, ART backlog |
| **Large Solution** | Multi-ART coordination, supplier integration | Solution train, solution backlog |
| **Portfolio** | Strategy, investment funding, governance | Portfolio kanban, strategic themes, OKRs |
