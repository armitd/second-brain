# Inter-Agent Communication Protocols: MCP and Alternatives

## Summary
Covers the emerging landscape of inter-agent communication protocols — MCP, ACP, A2A, and ANP. ACP focuses on local agent teamwork; A2A enables agents across different platforms to collaborate; ANP aims for a decentralised network of agents that share information securely without a central authority. Project BeeAI is a notable implementation referenced.

## Key Highlights
- **ACP (Agent Communication Protocol):** Local agent teamwork — agents on the same system or network coordinating tasks
- **A2A (Agent-to-Agent):** Cross-platform agent collaboration — enables agents from different vendors or services to work together
- **ANP (Agent Network Protocol):** Decentralised agent network — agents share information securely without requiring a central authority or orchestrator
- **Project BeeAI:** A concrete implementation in this space worth monitoring

## Why It Matters
Directly feeds the MCP Governance project. The protocol landscape is consolidating around MCP (tool/data access) and these emerging agent-to-agent standards. An EA governance framework needs to account for all four, not just MCP.
