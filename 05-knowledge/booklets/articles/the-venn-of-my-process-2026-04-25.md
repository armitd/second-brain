# The Venn of my process

## Quick Summary
Designer Itay Dreyfus describes how Claude Code removed the final blocker between his design thinking and shipping working products — he no longer needs to code traditionally. He presents a Venn diagram of his new identity: simultaneously designing and building, with AI handling the translation between the two. He's launching a workshop (buildthings.club) around this methodology.

## Key Insights
- **Design tooling has barely evolved since Photoshop** — prototyping is still a separate, manual switchboard-rewiring exercise. Figma didn't fundamentally change the workflow.
- **AI as skill-gap bridge, not replacement** — Dreyfus isn't a developer, but Claude Code lets him build working products anyway. The professional boundary between designer and builder is collapsing.
- **Rush to prototype beats extended planning** — his new process prioritises getting something working fast so clients can react to reality, not wireframes.
