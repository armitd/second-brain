# Thread: LLM Artifacts — Dynamic Knowledge Consumption from LLM Knowledge Bases

## Summary
Omar Sanseviero explores "LLM Artifacts" as a new medium for humans to consume LLM-generated insights from knowledge bases. Rather than static documents, artifacts are dynamic, interactive outputs built on top of LLM knowledge bases using tools like Chart.js and the Hacker News Firebase API. The frame is: artifacts aren't just AI outputs, they're a new category of interactive content.

## Key Points
- **Artifacts as a new medium:** LLM knowledge bases can generate dynamic, interactive content rather than just text responses
- **Chart.js + HN Firebase API:** Concrete tooling stack shown — real-time data + visualisation libraries
- **Human consumption layer:** Artifacts sit between raw LLM knowledge and end-user — they make knowledge accessible without requiring the user to prompt
- **Shifts the question:** from "how do I get the LLM to answer my question?" to "what is the best artifact to make this knowledge consumable?"

## Why It Matters
Relevant to how COG could evolve beyond text-based notes — interactive dashboards or visualisations built from vault knowledge. Also relevant for any project where knowledge needs to be surfaced to non-technical stakeholders.
