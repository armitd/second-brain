# Thread: 36 Obsidian Setups That Turn Messy Notes into Ideas, Income, and Momentum

## Summary

36 Obsidian workflows across 7 categories: capture, processing, output, client work, learning, AI automation, and monetisation. The governing philosophy: stop treating Obsidian as a filing cabinet; treat it as a production system where raw material (notes, ideas) gets processed into output (content, decisions, products).

## Key Points

**The core philosophy shift:** "If it only helps things go in, it's organisation. If it helps things come out, it's a money-making machine."

**7 categories:**
1. **Capture** — Three-inbox system (Ideas / Reference / Actions); voice-to-vault pipeline; meeting intelligence notes
2. **Processing** — Weekly processing ritual (60 min: process inboxes → flag #ready notes → turn one into output); maturity scale (#seed / #growing / #ready / #published)
3. **Output** — Content assembly line; personal framework builder; AI processing pass (weekly prompt: "what is this note actually arguing? what's missing? is this one idea or two?")
4. **Client/Business** — Client intelligence file (real goals vs. stated goals, exact language, what irritates them); pricing confidence note (log every exceptional result, read before pricing conversations)
5. **Learning** — Expert positioning map; learning-to-income pipeline; question bank (repeated questions = content calendar + course outline)
6. **AI Automation** — Claude as thinking partner (same conversation, never start new); weekly AI synthesis prompt; local Ollama for private notes (#local-only tag)
7. **Monetisation** — Vault-to-newsletter pipeline; launch command centre; referral intelligence tracker

**Best single setup:** The AI Processing Pass (Setup 9) — weekly prompt on 5 best notes: What is this actually arguing? What's missing? One idea or two? What would the title be?

## Why It Matters

Low direct relevance to Belron projects, but useful for optimising COG's own knowledge processing workflow. The "maturity scale" (seed/growing/ready/published) is a better status taxonomy than the current braindump flow. The AI Processing Pass prompt is directly usable on COG braindumps to surface what's ready for consolidation into 05-knowledge.
