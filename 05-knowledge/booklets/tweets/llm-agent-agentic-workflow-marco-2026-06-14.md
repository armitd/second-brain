# Thread: LLM vs Agent vs Agentic Workflow vs Multi-Agent System

**Author:** David Marco (@David_TornAI) — Jun 2026

## Summary
A clear 4-tier framework distinguishing the four levels of AI system design, with a key warning: most teams overbuild when a simpler tier would suffice.

## Key Points

| Tier | Mode | Autonomy | Best For |
|------|------|----------|----------|
| **LLM** | Generate | ★☆☆☆☆ | Writing, Q&A, summarisation, brainstorming |
| **Agent** | Act | ★★★☆☆ | Research, automation, troubleshooting, task completion |
| **Agentic Workflow** | Orchestrate | ★★★★☆ | Business ops, document processing, compliance, repetitive processes |
| **Multi-Agent System** | Collaborate | ★★★★★ | Large-scale projects, complex research, multi-stage enterprise workflows |

**The 1-line framework:**
- Need an answer? → LLM
- Need actions taken? → Agent
- Need consistency and control? → Agentic Workflow
- Need multiple specialists working together? → Multi-Agent System

**The anti-pattern**: "Many teams build multi-agent architectures for problems a single prompt could solve. More autonomy = more coordination, more latency, more cost, more failure points."

**The principle**: "The best AI architecture isn't the most advanced. It's the one that's just advanced enough for the problem you're solving."

## Why It Matters
Perfect stakeholder communication tool for MCP Governance conversations. When Belron colleagues use "AI agent" loosely, this framework gives you a shared vocabulary. Also directly useful for scoping the AI Damage Assessment PoC: which tier is actually needed for windscreen assessment? Likely an Agentic Workflow (structured process, predictable flow, human approval gate) rather than a full multi-agent system.
