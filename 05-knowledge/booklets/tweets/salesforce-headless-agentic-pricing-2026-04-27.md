# Thread: Salesforce Going Headless Is Bigger Than People Realise

## Summary
Salesforce announcing Salesforce Headless 360 — the entire platform (Salesforce + Agentforce + Slack) exposed as APIs, MCP, and CLI, with no browser required. The argument: this breaks the per-seat SaaS pricing model. Agents don't log in, they make API calls. Every SaaS company now faces an existential question about how to price for agentic usage.

## Key Points
- **Per-seat pricing is the wrong model for agents.** Software has been priced per seat for decades, assuming a human logs in. Agents make API calls — one company running 50 agents can make more API calls in a day than the whole sales team does in a month
- **Salesforce forced the question into the open.** Salesforce Headless 360: "Our API is the UI." Entire platform — data, workflows, tasks — accessible via API, MCP, and CLI
- **Agent-native pricing will define the next cycle.** Companies that figure out agentic pricing first will win; those still charging per seat while agents do the work will get left behind
- **MCP as the integration layer.** Salesforce exposed via MCP means Claude Code and other AI agents can interact with Salesforce natively — directly relevant to enterprise tooling governance

## Why It Matters
EA-relevant signal. As Belron evaluates agentic AI platforms, the pricing model question will become critical — especially for any tools that agents interact with rather than humans. MCP governance work should track which enterprise tools are exposing MCP endpoints (Salesforce now confirmed).
