# Thread: Build JARVIS in Obsidian With Claude Code — Full Setup From Scratch

## Summary
CyrilXBT's complete guide to building a JARVIS-style AI system inside Obsidian using Claude Code. The system does four things: captures thoughts with zero friction, automatically connects ideas across the vault, generates content briefs from connections, and writes in the user's voice. Full vault architecture, CLAUDE.md structure, and skills provided. Similar pattern to COG but with a content-creation focus.

## Key Points
- **Four capabilities:** capture → connect → brief → write (in your voice)
- **Vault architecture (JARVIS pattern):**
  - `00-INBOX/` — raw captures
  - `01-CAPTURES/` — typed captures (observations, reactions, patterns, questions, numbers)
  - `02-CONNECTIONS/` — AI-synthesised connections
  - `03-BRIEFS/` — content briefs from connections
  - `04-PUBLISHED/` — finished output
- **Requirements:** Obsidian (free), Claude Code (npm install), Anthropic API key — no paid plugins
- **CLAUDE.md is the brain:** detailed instruction file telling Claude the vault architecture, capture types, and how to process them
- **Skills:** process-captures skill, connection-finder skill, brief-generator skill, voice-writer skill
- **Gets smarter over time:** the more captures, the better the connection-finding becomes

## Why It Matters
A parallel implementation of the same pattern COG uses, oriented towards content creation rather than general second brain. The `01-CAPTURES/` typed capture taxonomy (observations/reactions/patterns/questions/numbers) is a useful schema that could inform how COG braindumps are tagged.
