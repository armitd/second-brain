# Thread: Obsidian for Beginners in 2026 — Everything You Need to Know

## Summary
Comprehensive written guide covering Obsidian from scratch: interface, note creation, Markdown, wikilinks, properties, Bases, templates, Graph view, syncing, web clipper, plugins, and themes. Positions Obsidian as the AI-native note-taking choice due to data sovereignty and direct file access by AI agents.

## Key Points
- **Data sovereignty argument:** Obsidian notes are plain Markdown files on your computer. No API overhead, no format conversion — an AI agent reads them directly like opening a text editor. The vault compounds.
- **Two power keys:** Quick Switcher (`Cmd+O`) and Command Palette (`Cmd+P`) handle almost all navigation
- **Bases** (new native Obsidian feature): database-style views of notes using frontmatter properties — replaces the need for Dataview plugin for most use cases
- **AI-ready by design:** With Obsidian, the AI knows your entire system the moment it opens the vault. More vault = more powerful and personal AI
- **Plugin philosophy:** Start with zero plugins. Add one only when you hit a specific friction point. Avoid plugin sprawl.
- Author has separate guides for Claude Code + Obsidian and full AI second brain setup

## Why It Matters
Useful reference if onboarding new COG users or explaining the Obsidian architecture choice. The Bases section is worth exploring — native database views without Dataview plugin dependency could simplify COG's structure.
