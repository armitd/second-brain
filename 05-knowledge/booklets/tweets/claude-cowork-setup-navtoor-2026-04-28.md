# Thread: How to Set Up Claude Cowork the Right Way — 5 Features, Context Files

## Summary
Nav Toor's setup guide for Claude Cowork, emphasising that Cowork covers ~60% of knowledge work if set up correctly. The core unlock: stop thinking about better prompts, start thinking about better files. Five features ranked by impact, with setup instructions for each.

## Key Points

**The five features (ranked by impact):**
1. **File System Access** — Claude reads/writes to a folder on your machine. No more export-upload-download loop. Point it at a "Claude Context" folder and it references your context mid-task automatically.
2. **AskUserQuestion** — Cowork asks clarifying questions instead of guessing and giving "polished garbage." Asking 3 questions upfront leads to far better output.
3. **Plugins** — Specialist packs for your specific role. Load once, available across sessions.
4. **Instructions** — Permanent memory that loads every session (equivalent to CLAUDE.md for Cowork). Set once, always active.
5. **Connectors** — Live integrations with Slack, Drive, Notion, 50+ tools.

**Context Files Strategy (the most important setup):**
Create a "Claude Context" folder with three files:
- `about-me.md` — who you are, role, what success looks like
- `brand-voice.md` — communication style, phrases, what sounds wrong, writing examples
- `working-style.md` — how you want Claude to behave (questions first? short/long outputs? file formats?)

**Loop that covers 60% of knowledge work:** Describe task → Cowork asks 3 questions → answer them → come back 20 minutes later to a finished document.

## Why It Matters
The context files strategy (about-me / brand-voice / working-style) directly mirrors COG's structure. The AskUserQuestion feature is Cowork's version of the clarifying question pattern already embedded in COG braindump. Good reference for how to introduce Cowork to non-technical Belron colleagues.
