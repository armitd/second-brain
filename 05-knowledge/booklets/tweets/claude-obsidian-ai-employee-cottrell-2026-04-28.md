# Thread: Claude + Obsidian = A True AI Employee — Business Operations System

## Summary
Fraser Cottrell's full business operating system built on Obsidian + Claude Cowork. Three-part system: (1) Knowledge Base in Obsidian with linked files — Memory, Client Roster, Action Tracker, Frameworks, Templates; (2) Automatic Memory Loop — call transcripts via Fathom → Zapier → Google Drive → Claude extracts decisions/actions/client updates → writes back to Obsidian; (3) Intelligence Layer — Claude Cowork reads vault + MCP connectors (Slack, Calendar, Drive) for cross-tool awareness.

## Key Points
- **Memory file = AI employee onboarding doc:** who you are, your business structure, processes, tools, communication style, goals — written once, referenced by every AI interaction
- **Linked Obsidian structure:** Client Roster, Action Tracker, Frameworks library, Templates folder — all linked to a Home page knowledge graph
- **Automatic memory loop:** Fathom transcribes calls → Zapier auto-drops to Drive → Cowork reads transcripts daily → extracts decisions, action items, client details → writes back to correct Obsidian files. "My system remembered a shipping decision I'd forgotten completely."
- **MCP as the keys:** Claude Cowork connects to Slack, Calendar, Gmail, Drive, ClickUp via MCP — reads vault + live tool data together
- **The distinction:** using AI vs having an AI employee. An employee remembers what was discussed. The system does.

## Why It Matters
The automatic memory loop (call transcription → extraction → vault update) is a pattern not yet in COG. Fathom + Zapier is the pipeline, but any transcription tool (Otter, Fireflies, Google Meet export) works. This is the "system that remembers meetings so you don't have to" pattern.
