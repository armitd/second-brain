# Thread: The Third Era of AI Software Development — Michael Truell (Cursor)

## Summary
Cursor CEO Michael Truell describes three eras of AI-assisted coding: Tab autocomplete (era 1) → synchronous agents in prompt-response loops (era 2) → autonomous cloud agents that run independently over hours and return reviewable artifacts (era 3). At Cursor, 35% of merged PRs are now written by cloud agents running on virtual machines.

## Key Points
- **Era 1:** Tab autocomplete — one keystroke at a time assistance
- **Era 2:** Synchronous agents — developer directs through prompt-response loops, stays in the loop
- **Era 3:** Cloud agents — run autonomously on VMs, handle large tasks over hours, return artifacts (logs, video, previews) rather than diffs
- 35% of Cursor's internal PRs now created by autonomous cloud agents
- Developer role shifts: from writing code → "building the factory that creates software" — providing direction, equipping agents with tools, reviewing artifacts
- Agents carry zero coordination overhead vs. human teammates

## Why It Matters
The three-era model is a useful framework for Belron stakeholder conversations about AI maturity. It contextualises where enterprise tooling sits today (era 2 transitioning to era 3) and what the governance requirements of era 3 look like — directly relevant to MCP Governance and CCOTF.
