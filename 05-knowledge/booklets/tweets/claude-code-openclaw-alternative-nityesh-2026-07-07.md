# Thread: Claude Code Is the OpenClaw Alternative You Already Have

## Summary
Nityesh Agarwal argues Claude Code already delivers what people loved about OpenClaw, often better — because both are just harnesses on top of AI models. Claude Code was marketed narrowly as a coding tool (a Cursor alternative), so few saw it as a general-purpose agent for non-programmers until Claude Cowork launched. Run Claude Code from your home folder with free access and you get the OpenClaw experience. (Consolidates the thread and the linked Every.to article of the same title.)

## Key Points
- The core distinction: **model vs. harness**. OpenClaw and Claude Code are both harnesses; the capability comes from the model underneath.
- Claude Code's "coding tool" positioning obscured its general-agent potential; Dan Shipper was early to reframe it as a tool for non-programmers.
- Cowork was the unlock for non-technical use; running from the home folder with broad access turns Claude Code into a personal-agent / "AI employee" platform.
- Author demos "AI employees" built directly on the Claude Code harness.

## Why It Matters
Direct validation of the COG approach: COG is exactly this pattern — Claude Code as a general-purpose harness over the vault, not a coding tool. Reinforces the model-vs-harness framing useful when explaining COG (and Claude Code as an enterprise agent surface) to Belron stakeholders.
