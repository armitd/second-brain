# Thread: 5 Practices for Making Claude + Obsidian Actually Work — YB

## Summary
YB (@yb_effect) shares 5 practices from 9+ months of daily Claude + Obsidian use. The core insight: the Karpathy LLM wiki is great for research projects, but a personal thinking tool requires actual personal context — you can't outsource your own life experiences to AI. The value compounds only if you keep the habit of capturing genuinely personal observations.

## Key Points

1. **Daily commonplace practice:** Capture messy, unstructured observations constantly. Voice transcribe — include ifs, buts, ums. The personal context is what makes the tool useful for you; you cannot automate this part. Context compounds after a few weeks of consistent habit.

2. **Andy Matuschak atomic notes:** Think in concepts, not facts/dates. Atomic notes as API endpoints you call when problem-solving. "Writing is thinking" (Paul Graham) — write for yourself, not for others.

3. **Intentional reading with Readwise:** Realised past "productive" content consumption (podcasts, Substack scrolling) was cope. Now extremely selective: scrolling = searching for potential reads only; reading time = highlighting obsessively. "Who gets access to my brain?" is the filter.

4. **Agentic constitution (.md):** A single file that acts as the portal between Obsidian and Claude. Contains: who you are, your big question, how you work, main goals, current problems, team context. Every skill/subroutine reads this first. Updated weekly as focus changes — think of it as briefing your team.

5. **Knowledge Work PRs:** Subroutines (= skills/agents) that scan recent notes, identify threads, draft atomic note updates, then take you through a 5-minute voice session to approve/tweak/reject. Like a code PR process: agents propose, human approves. One example: "Tend the Vault" — reviews all daily commonplace entries since last run.

## Why It Matters
The "agentic constitution" maps directly to COG's `MY-PROFILE.md` pattern. Knowledge Work PRs map to COG skills. The key differentiator from Karpathy's wiki: personal context cannot be synthesised — it must be captured. This is the human-in-the-loop that makes personal AI actually personal.
