# Thread: Stop (Over) Organizing Claude

## Summary
Ruben Hassid's counter-intuitive argument: most Claude setups are bloated with things you never actually use — prompt libraries, 40 skills used once, global instructions so long Claude ignores half. The real setup is one file, six habits, zero folders.

## The 20 Things to Delete
Prompt library, skills used once, "just in case" connectors, 31-page prompting guide, screenshot hacks never run, collected screenshots, forgotten subscriptions, folders inside folders, the "perfect setup" never started, chat history you'll "go back to," YouTube tutorials at 3% watched, custom styles never switched on, abandoned projects, global instructions too long, thinking on "high" for simple tasks, 6 connectors when you use only one, overlapping skills, organized Cowork folders never opened, screenshots of other people's prompts, plugin packs for one task.

## The 6 Things to Keep
1. **One about-me file.** One `.md` file: your role, tone, non-negotiables. Claude reads it once and stops sounding generic. Beats every saved prompt template.
2. **Give Claude the goal, not the steps.** "Reply to this so they say yes" — not "write the email in 4 paragraphs, friendly, no bullet points." The more you micromanage, the dumber the output.
3. **Make it ask you questions first.** Add "Before you answer, AskUserQuestion." Claude interviews you instead of guessing.
4. **Never trust the first answer.** Reply: "What's wrong with this? Now fix it." Claude critiquing Claude beats Claude on the first try.
5. **Start a fresh chat every time.** A 40-message chat drags 39 old messages into every reply — slower, more forgetful, more generic.
6. **Hand it the hardest task you have.** Most people give Claude 5-minute jobs. Give it the messy spreadsheet, the dense report, the thing you've been avoiding.

## Why It Matters
A useful calibration for COG's own skill and setup architecture. The "one about-me file beats every saved prompt template" observation is a direct argument for CLAUDE.md's identity-first approach. The "fresh chat for new tasks" principle is the same session hygiene rule in COG's CLAUDE.md. Useful to revisit when COG's skill library starts accumulating clutter.
