# Thread: Claude Setup, Skills, and Automations — Khairallah AL-Awady Series

## Summary
Four threads by @eng_khairallah1 covering Claude setup, Skills, workflows, and automations. The core argument across all four: most Claude users treat it as a chatbot and miss the configured, skills-based, automation-connected version that runs autonomously. The threads reference Boris Cherny (Claude Code creator) and cover 40 features, 30 Skills/workflows, 45 automations, and automated task management.

## Key Points
- **40 Claude features most users miss** — covers Custom Instructions (persistent system prompt), Projects (workspace with own system prompt + knowledge files), Extended Thinking, web search, conversation styles, keyboard shortcuts, simultaneous conversations. Setup is the gap, not intelligence.
- **30 Claude Skills and workflows** — Claude's Skills are "permanent instruction files that teach Claude exactly how to execute specific tasks." Key skills: PDF, DOCX, PPTX, XLSX processing; Frontend Design; Brand Guidelines; Obsidian Skills (Obsidian CEO-built). Key workflow patterns: Master Context File (everything Claude needs to know in one file, referenced by every Skill); CLAUDE.md for code projects; weekly feedback loop to update Skills from mistakes.
- **45 automations using Cowork and MCP servers, zero code** — Content automations (morning trend scanner via web search MCP, thread converter, content calendar), email automations (inbox prioritizer via Gmail connector), research (competitor monitor via web search MCP, legal document summarizer), file organization, business (weekly review, invoice generator, process documentation). Build 10 per weekend over a month.
- **MCP servers as the infrastructure layer** — Multiple automations rely on MCP servers: web search MCP for research and trend scanning; Gmail connector for email processing; Google Drive connector for file access. MCP is presented as the automation plumbing, not as a governance concern.

## Why It Matters
The practical MCP usage examples (web search MCP, Gmail MCP as connectors in Claude Cowork automations) show how non-technical users are already building multi-tool agent workflows without understanding the underlying protocol governance layer. This is the consumerisation pattern that makes MCP governance urgent at enterprise scale.
