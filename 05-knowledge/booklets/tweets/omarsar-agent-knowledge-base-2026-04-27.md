# Thread: Building a Personal Knowledge Base for AI Agents — Elvis (@omarsar0)

## Summary
DAIR.AI researcher describes building an automated research paper knowledge base on top of Karpathy's LLM Wiki pattern. Key extension: automated paper curation via a tuned skill, indexed with qmd CLI for semantic search, then fed into an interactive artifact generator via MCP tools inside an agent orchestrator. The knowledge layer and the agent layer are tightly integrated.

## Key Points
- Automated curation: tuned a skill to find high-signal research papers daily — no longer manually reviewing
- Indexed with qmd CLI (Tobi Lutke's tool) — markdown + metadata, optimised for semantic search
- LLM Wiki → interactive artifact generator: agents surface insights visually, dynamically, via MCP tooling
- "The research is only as good as the research questions. And the research questions are only as good as the insights the agents have access to." — the knowledge layer and the agent layer are mutually dependent
- Next frontier: making the knowledge base more actionable — the search problem, not the build problem

## Why It Matters
Extends Karpathy's LLM Wiki with concrete implementation detail: automated ingestion, semantic search, MCP integration. This is the architecture direction COG would move toward at scale — beyond manual braindumps toward automated curation pipelines.
