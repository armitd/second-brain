### Karpathy's 9 rules for building AI agents
**Source:** Readwise/Tweets/Tweets From Moysei.md (@0xMoysei)
Teaser re-share of Karpathy's "9 rules" — headline: *"stop writing prompts; if you're iterating on a single message at 3am you're still in the prompting era."* Points to the full list (already circulating). Relevant to agentic-AI / loop-engineering thinking.

### Claude Code is the OpenClaw alternative you already had
**Source:** Readwise/Tweets/Tweets From Nityesh.md (@nityeshaga)
Argues most people missed that Claude Code covers the same ground as OpenClaw hype. Relevant to the `openclaw-raspi` project framing.

### Set up 8 files once, then barely prompt again
**Source:** Readwise/Tweets/Tweets From Ruben Hassid.md (@rubenhassid)
Repeated theme across his tweets: build a context folder (about-me.md etc.) so Claude reads your identity/priorities before every task; "prompting is the worst way to use Claude." Also a "stop over-organising Claude — delete your prompt library, unused skills, just-in-case connectors" take. Mirrors COG's `MY-PROFILE`/`MY-PREFERENCES` approach.

### Build 5 assistants in an afternoon / Google's 8-min AI agent lesson
**Source:** Readwise/Tweets/Tweets from Anatoli Kopadze.md (@AnatoliKopadze)
Two pointers: an Anthropic-engineer walkthrough on building small task assistants, and a free Google 8-minute lesson on AI agents and loops. Useful AI-literacy teaching material.

### "Anthropic shows this video to every new employee"
**Source:** Readwise/Tweets/Tweets From Raytar.md (@Raytar)
Re-upload of a ~14-min internal-style video on how the Claude team uses Claude in real work. Worth watching for practical usage patterns.

### NotebookLM + Claude; Tina Huang Cowork tutorial
**Source:** Readwise/Tweets/Tweets From Olivia Chowdhury.md (@Oliviacoder1)
"Ingest 50 PDFs with NotebookLM, think with Claude" — 8 prompts to compress research; plus a recommended Claude Cowork tutorial by Tina Huang (ex-Meta data scientist).

### STORM method — ask dozens of questions, not one
**Source:** Readwise/Tweets/Tweets From Riley West.md (@rileywestreel)
Cites Stanford's Monica Lam: AI writes shallow reports because you ask one question; their method interviews the model like a journalist (dozens of questions) and produces articles ~25% better organised. Directly relevant to the `auto-research` / `deep-research` skills.

### Raw Claude Code is only ~20% of the product
**Source:** Readwise/Tweets/Tweets From Gipp 🦅.md (@gippp69)
A 41,000-star GitHub repo used to argue the biggest Claude Code upgrades aren't new features but community configs/skills/context — the base tool is ~20% of the real product. AI-literacy point aligned with the "set up your context once" theme above.
