# Thread: The Complete Claude Tools Guide — Four Tiers from Projects to Coding

## Summary
Four-tier framework for Claude tools organised by productivity impact. The author argues the biggest unlock wasn't advanced prompting or Claude Code — it was using the built-in tools that 99% of users ignore. Projects and Skills together are described as the foundation everything else runs on.

## Key Points

**Tier 1 — Core Interface Tools (highest impact)**
- **Projects:** Single most important feature most people haven't properly set up. Without Projects, every conversation starts from zero — Claude knows nothing about you. Spend 20-30 minutes building context files, upload to a dedicated Project. Instantly better responses.
- **Skills:** Reusable instruction sets in markdown files. If Projects are the environment, Skills are the workflows inside them. Example: Brand Voice Skill (tone, audience, banned words, formatting rules) — call it instead of re-explaining every time.

**Tier 2 — Research & Thinking Tools**
(covers Claude's built-in research, analysis, and reasoning capabilities)

**Tier 3 — Agentic Tools**
(multi-step, autonomous task execution)

**Tier 4 — Coding & Building**
(Claude Code, development workflows)

**Core insight:** The biggest productivity gains come from foundational setup (Projects + context files), not from sophisticated prompts or agentic workflows.

## Why It Matters
Validates the COG architecture: context files + skills as the foundation, with agentic capabilities layered on top. The four-tier framing is a useful mental model for explaining to colleagues where to start with Claude.
