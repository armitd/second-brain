# Thread: Dami-Defi — Claude + Obsidian 4-Layer System

**Author:** Dami-Defi (@DamiDefi) — May 2026

## Summary
A practitioner account of running Claude as the intelligence layer over an Obsidian vault for two months. The core finding: the difference between an organised vault and an intelligent one is the difference between a library and a researcher. The system is built on four layers; remove any one and it degrades into a fancy note-taking app.

## The Four Layers

### Layer 1: Capture (Zero Friction)
- **Readwise** handles all external content — every highlight from articles, books, PDFs, newsletters flows automatically into Obsidian via Readwise Official plugin. No manual effort.
- Principle: the system dies if capture requires effort. Every time you think about where something goes, you burn willpower that should go toward thinking about the content.

### Layer 2: Storage (Structured but Not Rigid)
- Obsidian vault with consistent structure — not elaborate, but consistent enough that Claude can navigate it reliably
- Organisation is just a faster way to find something you already know exists. It does not create new knowledge.

### Layer 3: Intelligence (Claude via Filesystem MCP)
- Claude connects to the vault via Filesystem MCP — can read every note, reason across all of them simultaneously
- Every morning before opening anything: Claude has already been through recent captures and produced a synthesis. Not a summary — connections missed, patterns forming across weeks, questions worth thinking about that day.

### Layer 4: Output
- The synthesis outputs become posts, decisions, frameworks — evidence the vault is producing rather than accumulating

## Why It Matters
This is a practitioner confirmation of the COG architecture. Specifically: **Readwise → Obsidian → Claude MCP** is validated as a working system by someone running it in production for 2 months. The "library vs researcher" framing is the cleanest articulation of why COG exists — not to store more, but to think better. Directly supports the AI Damage Assessment PoC positioning around AI as analyst, not just database.

The key design constraint: Layer 1 (Readwise zero-friction capture) must be maintained. Any friction in capture kills the compound effect in Layers 3 and 4.
