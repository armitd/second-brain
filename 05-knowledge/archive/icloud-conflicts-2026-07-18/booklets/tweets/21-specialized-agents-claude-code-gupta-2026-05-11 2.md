# Thread: 21 Specialized Agents Beats One Prompt — Aakash Gupta

## Summary
Aakash Gupta argues that multi-agent specialisation in Claude Code mirrors how the brain organises cognition: dedicated regions outperform general-purpose ones because each gets clean signal without noise from adjacent tasks. One prompt trying to do everything produces AI slop; 21 agents each with one job produce a shippable product.

## Key Points
- The brain refuses to let one region handle both face recognition and speech comprehension — not for lack of space, but for fidelity. A region doing two jobs averages the signal until both degrade.
- In Claude Code, the same principle applies: system analyst, test architect, and code maintainability agent cannot share a context window. Each needs isolation to do its job well.
- Gabor's 21-agent system produced a Flutter + Firebase + Vertex DB hockey rules AI app — uploaded to TestFlight in one podcast session, using 10% of a $200 Claude Code plan.
- The 21-agent workflow: system analyst writes Confluence docs → agents file 51 Jira tickets in sprints with dependencies → Figma agents drive screens through MCP → UX flow agent wires prototype arrows. No context window ever sees another's noise.
- "The PMs writing one prompt and getting AI slop are asking the same neuron to recognise their mother and parse a sentence."

## Why It Matters
Directly relevant to MCP Governance: the specialisation argument supports designing Belron's agentic architecture as purpose-built agents per domain (damage assessment, booking, ADAS routing) rather than one generalised assistant. Also relevant to Contact Centre of the Future — specialised agents per task type (authentication, billing, refund) outperform general-purpose chatbots.
