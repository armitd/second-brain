# Thread: How to Give Claude Perfect Memory — Complete Guide

## Summary
A three-layer Claude memory system: (1) direct prompting — explicit context in conversation; (2) import/export context — structured prompts that preserve state across sessions; (3) deep wiki system using Obsidian — Claude builds, updates, and links notes automatically, creating an evolving knowledge base. The guide argues that without a memory system, Claude's context amnesia is a fundamental productivity bottleneck.

## Key Points
- **Layer 1 — Direct prompts:** Tell Claude explicitly what to remember within a session; use structured context blocks at conversation start
- **Layer 2 — Import/export context:** End sessions with "summarise key decisions for next session" style prompts; start new sessions with that summary
- **Layer 3 — Wiki system:** Obsidian + Claude Code builds a persistent, auto-linking knowledge base; Claude reads and writes notes so knowledge accumulates across sessions and conversations
- Layer 3 is the architecture COG uses — this guide validates and provides comparison reference for alternatives

## Why It Matters
COG's CLAUDE.md + memory system + skills is a Layer 3 implementation. This guide provides an external reference point for the same architecture, useful for explaining COG to new users or for identifying any memory layer gaps to improve.
