# Thread: Codex Knowledge Vault — Automated, Self-Updating — Ziwen / CyrilXBT

## Summary
Ziwen describes "Codex" — an automated personal knowledge system built on Obsidian where every X bookmark is automatically scraped, processed, and stored overnight using Claude + computer use + N8N. The vault evolves daily without manual input. The key insight: organise for the machine, not the human, using a flat 5-layer architecture with an AGENTS.md master file as the AI's persistent context.

## Key Points
- **Context Debt** is the core problem: every new chat starts cold because AI doesn't know your history. Re-explaining your stack every morning is a tax on productivity. A persistent knowledge layer eliminates it.
- **5-layer flat architecture:** AGENTS.md (global variable — who you are, 2026 goals, non-negotiable rules) → inbox/ (raw staging, no tagging) → notes/ (processed, interlinked facts — "personal Wikipedia") → ideas/ (your original thinking, prevents generic AI answers) → projects/ (active work, adjacent to notes so agents can cross-reference).
- **Passive ingest via computer use:** Browser-use or Computer Use in Codex scrapes X bookmarks daily without manual effort. No new habits required.
- **The Freshman Rule:** agents must cite a specific vault file before making a technical decision; must write a 3-sentence "Battle Plan" before writing code; must stop and ask if a task contradicts an older note.
- **Weekly firmware upgrade:** rewrite the AGENTS.md "Core Logic" section every Monday to integrate the week's learnings — the agent starts Monday as "a more capable, more senior version of itself."

## Why It Matters
Alternative architecture to COG — Codex uses computer use for passive ingest vs COG's Readwise integration. The AGENTS.md "global variable" pattern and the Freshman Rule (cite before claiming) are both worth considering as COG governance patterns. The weekly AGENTS.md rewrite is an interesting self-improvement loop.
