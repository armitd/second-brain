# Thread: Claude Code + Obsidian Ultimate Guide

## Summary
A practical guide to combining Claude Code with Obsidian to build an AI second brain — directly inspired by Karpathy's LLM Wiki pattern. The system: Claude Code reads, summarises, links, and updates a wiki automatically from raw source documents; it can answer complex questions across the entire knowledge base, build backlinks, and improve over time without manual editing. The guide claims a 5-minute setup and a built-in memory system that improves with use.

## Key Points
- Claude Code's file-system access is the key enabler — it can read the entire vault, not just what you paste into a chat window
- The pattern: `raw/` folder for source documents → Claude compiles to a wiki of interlinked `.md` files → LLM answers questions across the full knowledge base
- Obsidian Web Clipper + local image download is the ingest pathway for web articles
- The guide credits Karpathy's LLM Wiki as its inspiration — the same pattern underlying COG

## Why It Matters
Directly relevant to COG's own architecture — this is a public tutorial of the same pattern COG implements. Useful for understanding what others are building on the same stack and for identifying features or workflows that COG could adopt or that validate its current approach.
