## High Impact News

### ServiceNow Launches Open MCP Governance Platform at Knowledge 2026
**Relevance:** This is the closest thing to a production blueprint for your MCP Governance project — an enterprise-grade, GA reference implementation of MCP with audit trails, OAuth, RBAC, and cross-vendor agent governance.

ServiceNow announced at Knowledge 2026 (May 5) that its platform is now open to any AI agent — Claude, Copilot, or customer-built — through a generally available MCP Server included in every Now Assist and AI Native SKU. The centrepiece is **AI Control Tower**, which discovers, observes, governs, secures, and measures AI deployed across any system in the enterprise. Alongside it, **Action Fabric** ships as an open MCP server, letting agents execute governed workflows (flows, playbooks, approvals, catalogs) with every action being identity-verified, permission-scoped, and fully audited. A new **AI Gateway** provides real-time controls for agentic workloads across third-party systems.

The A2A protocol (Agent2Agent) is embedded in ServiceNow's AI Agent Fabric, enabling cross-vendor agent collaboration without human re-prompting at each step. The MCP Server Console includes governance, consumption metering, managed OAuth, enterprise audit trails, session management, and role-based tool packages. Additional features are planned for 2H 2026.

**Impact Assessment:**
- **Projects Affected:** MCP Governance directly. This gives you a live enterprise reference for what governed MCP looks like in production.
- **Potential Effects:** If Belron is considering ServiceNow as a platform (or already uses it), MCP governance may be partially solved at the platform layer — which shifts your EA role from building governance to defining the cross-platform standards ServiceNow doesn't cover.
- **Action Suggested:** Review the AI Control Tower feature set against your MCP Governance framework. Identify which governance controls you were planning to build that ServiceNow now provides, and which gaps remain (non-ServiceNow systems, Belron-specific agent patterns).

## Strategic Developments

### A2A Protocol v1.2 Reaches 150 Organisations in Production
**Relevance:** The protocol stack underpinning your MCP Governance project is maturing faster than expected — A2A is now in real production deployments, not pilots.

Google's Agent2Agent (A2A) protocol has crossed into production at 150 organisations, governed by the Linux Foundation's Agentic AI Foundation. Version 1.2 introduces signed agent cards using cryptographic signatures for domain verification — meaning agents can now verify the identity of the agents they're collaborating with. Microsoft, AWS, Salesforce, SAP, and ServiceNow are all running A2A in production environments.

The relationship between MCP and A2A is now well-established: MCP handles how an agent connects to tools and data sources; A2A handles how agents communicate with each other across organisational and platform boundaries. This two-protocol stack is becoming the de facto enterprise standard.

**Strategic Implications:**
- Your MCP Governance framework needs to cover both protocols — MCP for tool/data access governance, A2A for inter-agent identity and trust.
- The cryptographic agent card standard in A2A v1.2 is directly relevant to any Belron multi-agent architecture. Agents from different vendors can now be verified before they're granted access to Belron systems.
- With ServiceNow, Microsoft, Salesforce, and SAP all in production on A2A, the protocol is no longer experimental. EA policy should treat it as the standard, not a future consideration.
